## 0.1) Exit Criteria

Use these as the minimum bar before a family can move from `substantive open gap` to `at parity in current gates`.

- Long-horizon execution:
  - a direct family proves 3+ later-turn continuation with accurate deferred-queue state
  - one family proves correct stop/go judgment when widening is tempting but not justified
  - one family proves explicit phase carry-forward across session restart or equivalent restored state
- Documents:
  - direct packs exist for architecture, incident, product/program, and operations documents
  - outputs are grounded in repo/runtime evidence, not generic templates
  - edits to existing long docs preserve structure, tone, and internal consistency
- Commands and system manipulation:
  - direct packs exist for long-running commands, service management, installs/upgrades, rollback, and environment diagnosis
  - summaries include concrete command evidence and rollback state
- Remote operations:
  - direct packs exist for single-host, multi-host, deployment, log/service diagnosis, and config-edit flows
  - summaries remain secret-safe and host-specific
- GUI and visual intelligence:
  - direct screenshot-backed gates exist for arbitrary GUI workflows, not only current site-specific flows
  - the system can detect and report visual layout faults from screenshots with actionable precision
- Multi-tool orchestration:
  - direct packs require coordinated use of multiple tool classes and coherent merged output
  - failure recovery across tools is explicitly tested
- Knowledge freshness and grounding:
  - direct freshness-sensitive families exist where external current facts matter
  - answers combine repo, runtime, and external evidence coherently when required

## 0.2) Validation Ladder

- `scaffolded probe`: targeted row or narrow family, used to expose the raw gap
- `default-pack row`: promoted into an existing default pack
- `family gate`: dedicated runner/pack with regressions, holdout, and transfer
- `release enforcement`: wired into release/soak defaults with fresh authoritative artifacts

The goal is not to stop at the first green probe. A family is only considered substantively closed once it reaches at least `family gate`, and operationally closed once it reaches `release enforcement`.
