# Artificer Perfection Checklist

Last updated: 2026-03-23

This checklist is the stable roadmap toward three targets:
- Fully usable every day with no message-loss or confusing flow.
- High reliability under adversarial, ambiguous, cross-domain tasks.
- Intelligence quality that is consistently competitive with top-tier assistant behavior.

## 1) UX And Flow

- [x] Add a dedicated `GUI Testing` run mode with strict hands-on UX-flow policy (status clarity, ordering coherence, visual artifacts, and interaction regressions).
- [x] Team profiles are global and selectable directly from the run-mode menu (`Team -> <team name>`), with a default-team picker in Settings.
- [x] Thread selection is not hijacked during draft-to-thread send.
- [x] Newly created threads do not disappear on stale state refreshes.
- [x] Thread selection routing keeps app pathname stable across reloads (hash route state), preventing reload-to-404 thread loss.
- [x] Command-approval inline does not stay stuck after backend clear/terminal queue status (no stale historical run-event bleed-through).
- [x] Failed draft sends restore prompt text across workspace/thread drafts.
- [x] No dead-air intervals: every run always has a clear live status until final answer appears.
- [x] End-to-end flow polish pass across all primary user journeys (new thread, switch, send, edit queued, decision, approval, archive).
  Progress: Added queue-edit post-save UX hold in app state/queue worker gating so edited items remain visible briefly before auto-resume, plus stronger Safari queue-edit journey binding (conversation-scoped queue item IDs, deterministic queue enqueue, and queue-take delay for the edit path). Added deterministic draft-hydration fixture hardening (conversation-scoped draft seeding + hydration recovery retries). Added queue-tray opportunistic queue probes so queue-edit controls can surface from real queue state even when lightweight conversation counters are briefly stale. Fresh Safari regressions are now green across all profiles: core (`gui-queue-edit-coherence-v1`), deep (`gui-queue-edit-coherence-v1-deep`), background (`gui-background-polish-v1`), and full (`gui-full-polish-v1`).

## 2) Message And State Reliability

- [x] Pending outgoing messages survive reload/restart.
- [x] Active conversation selection survives transient stale `state` payloads that temporarily omit the active thread summary.
- [x] Pending sends are persisted before enqueue completion (crash-window hardening right after submit).
- [x] Pending outgoing entries are pruned and bounded to avoid unbounded growth.
- [x] Pending outgoing entries reconcile automatically (including background-completed threads not reopened by the user).
- [x] Pending outgoing reconciliation self-heals during long busy periods and clears orphaned conversation keys.
- [x] Rapid multi-send from workspace draft reuses one created thread and preserves send order deterministically.
- [x] Add explicit crash/restart recovery fixture: submit -> forced reload -> confirm pending and recovery semantics.
- [x] Add deterministic fixture for rapid multi-send ordering and dedupe behavior.
- [x] Active workspace selection deprioritizes unusable/stale entries (missing path or sentinel IDs like `null`), with automatic re-selection to a valid workspace.
  Progress: `state?level=light` now reports true filesystem-backed `path_exists` values, run-path checks no longer auto-create missing workspace directories, and frontend selection/sorting now prefers usable workspaces plus auto-refreshes state after workspace-path run failures.
